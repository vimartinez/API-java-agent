# Especificaciones (spec)

Coloca aquí los archivos Markdown que definen:

- **Requisitos** de negocio o de producto.
- **Contratos de API** (endpoints, request/response, códigos de estado).
- **Decisiones de diseño** (arquitectura, convenciones, integraciones).

El agente de Cursor usa `AGENTS.md` y las reglas en `.cursor/rules/` como contexto; puedes referenciar estos spec desde ahí o mencionarlos en el chat con `@docs/spec/nombre-del-spec.md`.

## Ejemplo de estructura

- `api-contract.md` — Contrato REST (recursos, métodos, DTOs).
- `requirements.md` — User stories o requisitos funcionales.
- `architecture.md` — Diagramas y decisiones técnicas.
